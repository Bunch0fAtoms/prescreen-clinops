The Foundation: Unity Catalog for AI

Unity AI Gateway extends Unity Catalog's governance model to AI workloads. Models, agents, and tools become first-class securable objects alongside your tables and volumes.

Unified Permissions

Grant EXECUTE on a model just like SELECT on a table. The same role-based access control (RBAC) and attribute-based access control (ABAC) patterns apply. One governance model for all assets.

Complete Lineage

Track data flow from source tables through models to predictions. Every transformation is recorded, including which model version processed which data.

Key insight: You don't need to learn new governance patterns. The Unity Catalog skills your team already has now apply to AI. The same audit queries that work for table access work for model usage.

Unity AI Gateway Capabilities

The Beta release brings enterprise-grade governance to AI workloads. These capabilities are live in your workspace today.

Service Policies

Control requests based on content, user, and context. Allow, deny, or require approval for specific operations.

→

Example in ML Session

Policy blocks any attempt to export patient names from ai_query, while allowing biomarker extraction.

Cost Management

Set budgets at user, team, and workspace levels. Hard caps stop requests when limits are reached.

→

Example in ML Session

Set a monthly budget for the ML team, then watch spend accrue per step (extraction, embeddings) against it in near real time. The figures shown in the demo are illustrative.

Model Context Protocol (MCP) Governance

Managed connectors for tools like Slack, Jira, and GitHub. Agents inherit user permissions automatically.

→

Example in ML Session

Genie can only query tables the user has SELECT on. No elevation of privilege.

Inference Tables

All predictions logged to Delta tables. Query with SQL for audit and analysis.

→

Example in ML Session

Every coordinator's eligibility check is logged with timestamp, user, and decision.

Live Demo Flow

During the ML session, we'll open the Unity AI Gateway UI to show these governance features in action.

1. Show Registered Models

Navigate to Unity AI Gateway (Beta) in the sidebar. Display registered models:

  • databricks-claude-haiku-4-5, a managed foundation model
  • clinicalbert_note_embedder, a custom registered model
  • Permissions tab showing GRANTs

2. Review Service Policy

Policy: PHI Protection (illustrative) Condition: response matches a sensitive pattern (for example a social security number or date of birth) Action: DENY Message: "Protected health information blocked"

Show how a guardrail policy protects against accidental PHI leakage while allowing legitimate biomarker extraction. The exact policy syntax and available conditions follow your workspace's AI Gateway guardrail configuration.

3. Cost Dashboard

Display real-time usage metrics:

  • Cost breakdown by model and user
  • Token usage trends
  • Budget utilization (e.g., "$127 of $500 used this month")
  • Alerts configured at 80% threshold

4. Query Inference Logs

SELECT user_name, model_name, COUNT(*) as queries, SUM(token_count) as total_tokens, SUM(estimated_cost) as total_cost FROM -- where AI Gateway payload/usage logging lands WHERE date >= current_date() - 7 GROUP BY user_name, model_name ORDER BY total_cost DESC

Demonstrate the audit capability with a live SQL query over the past week's usage. Point the query at the inference or payload log your AI Gateway endpoints write to, or the relevant serving system table in your workspace.

The Value Story

Unity AI Gateway gives you production-ready AI governance using the patterns you already know.

For Data Governance Teams

  • Same GRANT/REVOKE patterns extend to AI
  • Audit logs capture every model interaction
  • Service policies enforce compliance rules
  • Cost allocation enables chargebacks

For ML Teams

  • Models become first-class UC objects
  • Automatic lineage tracking
  • Built-in cost controls prevent overruns
  • Production deployment with governance built in
The key message: Unity AI Gateway isn't a new governance system to learn. It extends the Unity Catalog governance you already trust to cover AI workloads. Your existing skills, patterns, and policies now apply to models, agents, and AI tools. This is how you scale AI adoption with control.
