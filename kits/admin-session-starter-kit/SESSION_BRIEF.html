2 How the session works

The hard math is done. You own the questions.

This is a lighter kit than the build sessions: a guided runbook, ready prompts, and working SQL Amy can run today and schedule tomorrow. Not a heavy notebook bundle.

  • The usage ↔ list_prices join, using the price in effect at usage time.
  • The chargeback category logic — department tag, product fallback, DE / DS / Research buckets.

Validated on a live workspace

Both queries ran against live system.billing tables and returned correct month-over-month chargeback breakdowns and budget-status flags. The SQL is the trusted version Genie One is approximating.

3 The flow

From a plain question to a scheduled answer

Three steps, all in Genie One, then an optional self-serve space for the wider team.

step 1

Ask

Open Genie One, point it at the billing system tables, and ask a cost question from the prompt sheet. Each prompt has a one-line "what good looks like" so Amy knows the answer is right.

step 2

Trust

Add the validated cost_by_category.sql as a trusted sample query, so Genie One's answers sharpen and stay consistent.

step 3

Schedule

Stand up the monthly report and the budget alert. The right primitive is a SQL Alert or dashboard subscription, since job emails carry run status, not query results.

An optional next step: a self-serve cost Genie space, so finance and team leads can ask their own spend questions in plain language.

4 Ground rules

Read-only, governed, no patient data

The same governance spine as every other session, applied to billing telemetry.

Read-only

Everything reads system.billing. The queries write nothing. Any scratch table lands in an admin schema, never in system.

No PHI

Billing tables hold usage, SKUs, tags, and dollars. No patient data. This session never touches a clinical schema.

Unity-Catalog-scoped

system.billing is governed by Unity Catalog. An account admin enables it and grants access.

Works even without live billing

If billing is not yet enabled, the queries are still correct. Point them at a small synthetic fallback to demo the shape and Genie One's behavior.
